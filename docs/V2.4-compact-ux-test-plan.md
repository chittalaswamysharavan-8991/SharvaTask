# SharvaTask MCP V2.4 — Compact UX + Testing

## User stories

1. As a mobile ChatGPT user, I want the widget to show the current list in a compact card so it does not occupy the whole screen.
2. As a task user, I want to add a task quickly without a tall form.
3. As a task user, I want to edit a task title, note, and priority from the widget.
4. As a task user, I want to mark done, reopen, block, and edit without leaving the widget.
5. As a user checking history, I want readable history events instead of raw JSON.
6. As a user browsing lists, I want only the latest few lists first, with Show more when needed.
7. As a mobile user, I want compact icon actions for history/refresh/archive.
8. As a desktop user, I still want the widget to stay clear and readable.

## V2.4 changes

- Added `update_task` MCP tool.
- Added `task_updated` event.
- Materializer now updates task title, notes, and priority.
- Widget now includes an Edit button for tasks.
- History widget now shows human-readable event cards; raw payload is hidden under Raw event.
- Mobile CSS was compacted:
  - Header actions become icon buttons.
  - Stats stay in one compact row.
  - Add task composer stays in one compact row.
  - Task cards are shorter.
  - List cards are shorter.
- List browser initially shows 5 lists and has Show more.
